Yes, forcing users to login before filling a form is the **perfect solution**. It ensures 100% data consistency.

By doing this, every single submission is guaranteed to be linked to a real User ID in your database. You will never have "orphan" emails.

### 1. How to Enforce Login

You don't need new code. You just need to check the `requiresLogin` flag in your **Form Model** (which we added earlier).

* **Logic:** When you create a form, set `requiresLogin: true`.
* **Frontend Behavior:** When a student opens the link `ieee.org/forms/123`:
1. Frontend checks `form.requiresLogin`.
2. If `true` AND user is not logged in → Redirect to `/login`.
3. User logs in → Redirect back to `/forms/123`.

### 2. Full API Documentation (So Far)

Here is the reference manual for the functions we have built. You can give this to your Frontend Team.

#### **Base URL:** `http://localhost:5000/api`

---

### **A. Authentication Module**

| Endpoint | Method | Access | Description |
| --- | --- | --- | --- |
| **/users/login** | `POST` | Public | Authenticates user & sets HttpOnly Cookie. |
| **/users/logout** | `POST` | Public | Clears the auth cookie. |
| **/users** | `POST` | Public | Registers a new user (Default role: `user`). |
| **/users/profile** | `GET` | Private | Returns current user info (checks cookie). |

**1. Login Request**

* **URL:** `/api/users/login`
* **Body:**
```json
{
  "email": "admin@ieee.org",
  "password": "password123"
}

```


* **Response (200 OK):**
```json
{
  "_id": "65b...",
  "name": "IEEE Tech Head",
  "email": "admin@ieee.org",
  "role": "admin"
  // Note: Token is NOT here. It is in the browser cookies.
}

```



**2. Register Request**

* **URL:** `/api/users`
* **Body:**
```json
{
  "name": "New Student",
  "email": "student@uni.edu",
  "password": "securePass123"
}

```



---

### **B. User Management & Filtering (The Email List Source)**

| Endpoint | Method | Access | Description |
| --- | --- | --- | --- |
| **/users** | `GET` | Admin | Fetches users with advanced filtering/sorting. |

**Advanced Filter Examples (Query Strings)**

* **Search by Name/Email:**
`GET /api/users?search=Ahmed`
* **Filter by Role:**
`GET /api/users?role=member,board`
* **Filter by Join Date (This Year):**
`GET /api/users?startDate=2026-01-01&endDate=2026-12-31`
* **Get Event Attendees (Who actually scanned QR):**
`GET /api/users?formId=65a9f...&attendedOnly=true`
* **Get Event Registrants (Everyone who signed up):**
`GET /api/users?formId=65a9f...`
* **Pagination & Sorting:**
`GET /api/users?page=1&limit=50&sort=-createdAt`

**Response Structure:**

```json
{
  "users": [ ... array of user objects ... ],
  "total": 150,
  "page": 1,
  "pages": 3
}

```

### **C. Form Module**

| Endpoint | Method | Access | Description |
| --- | --- | --- | --- |
| **/forms** | `POST` | Admin | Saves a new form structure. |
| **/forms/:id** | `GET` | Public | Fetches form structure (questions) to render. |

**1. Create Form Request**

* **URL:** `/api/forms`
* **Headers:** `Authorization: Bearer <token>` (Handled by Cookie usually)
* **Body:**
```json
{
  "title": "Cybersecurity Workshop",
  "description": "Learn how to hack.",
  "type": "event",  // Options: 'general' or 'event'
  "structure": [
    // This JSON comes directly from the React Form Builder library
    { "id": "field_1", "label": "Phone Number", "type": "text" }
  ],
  "settings": {
    "requiresLogin": true, // <--- The logic you asked for
    "maxSubmissions": 100,
    "expiryDate": "2026-02-20T00:00:00.000Z"
  }
}

```



**2. Get Form Request**

* **URL:** `/api/forms/65b92...`
* **Response:** Returns the JSON object above.
