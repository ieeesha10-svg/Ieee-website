Contributing

Thank you for contributing. Follow these guidelines to ensure a smooth review process.

Workflow
- Fork or branch from `main`.
- Open a feature branch: `feature/brief-description` or `fix/brief-description`.
- Open a PR with a clear title and description.

Pull request checklist
- [ ] Branch has a descriptive name
- [ ] Code builds and runs locally
- [ ] Linting passes
- [ ] Tests (if any) added/updated
- [ ] PR description lists what changed and why

Commit messages
- Use clear, present-tense messages. Example: `Add event list endpoint`

Code review
- Assign reviewers, address comments, and squash/fixup commits as requested.
