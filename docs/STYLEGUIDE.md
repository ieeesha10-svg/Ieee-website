Style Guide

Frontend
- Use functional React components and hooks.
- Keep components small and focused; separate presentational and container logic.
- Prefer CSS Modules or Tailwind utilities depending on team preference.

Backend
- Keep controllers thin; move business logic to services.
- Use consistent error handling and HTTP status codes.

General
- Use ESLint and Prettier. Add configuration files in the repo if not present.
- Write descriptive variable and function names.
